This repository accompanies the manuscript:

> Parras-Moltó M, García-Ríos E. *Triangulating compound effects in
> 96-well growth-curve phenotypic screens: a benchmarked,
> simulation-validated pipeline combining within-plate B-score
> correction, median-of-controls normalisation and linear mixed
> modelling*. **PLOS Computational Biology** (under review, 2026).
> doi:10.5281/zenodo.20702473.

It contains all the analysis code required to regenerate every
numerical result and every figure of the paper. The raw OD600
recordings are available on request from the corresponding author;
all intermediate tables are produced deterministically by running
the deposited scripts. The final figures and the typeset manuscript
are not duplicated here — the paper is served by PLOS Computational
Biology and the production-quality figures are attached to the same
Zenodo concept record as this repository.

The reusable analysis pipeline is distributed separately as the R
package [`gcscreen`][gcscreen]
(doi:10.5281/zenodo.20702376).

[gcscreen]: https://github.com/mparmol/gcscreen

## Summary of the study

A 1,513-compound phenotypic screen of the Prestwick Chemical
Library against *Saccharomyces cerevisiae* in 96-well plates with
high-resolution OD600 kinetics over 60 hours. The contribution is
a benchmarked, simulation-validated pipeline for kinetic 96-well
screens that combines plate-level quality control, within-plate
B-score positional correction, median-of-controls normalisation,
per-plate ANOVA and a library-wide linear mixed model with plate
identity as a random effect, requiring convergence between two
inferential frameworks before calling a hit. Eighteen reproducible
core hits are reported as an applied case study.

## Raw data — available on request

The raw OD600 kinetic recordings of the screen (57 plate-runs,
~5,500 wells, ~60 hours per recording; ~3.4 MB as a single zip)
are **not deposited in this public repository**. They are
available **on reasonable request** from the corresponding author:

- **Marcos Parras-Moltó** — mparmol@gmail.com

Requests are answered within a few working days. The dataset is
shared as a single zip under the CC-BY 4.0 licence (see
`LICENSE-data`).

Once obtained, place the CSV files under `data/raw/` at the
repository root, with the filename pattern
`plate_<plate>_rep_<replicate>.csv` where `<plate>` runs from 2 to
19 (plates 1 and 18 were excluded *a priori* on the basis of
within-plate control CV > 45%) and `<replicate>` from 1 to 4.
Each file has time in column 1 (hours since assay start) and 96
OD600 columns named by well (`A1`, `A2`, …, `H12`).

## Intermediate tables — regenerated by the scripts

The intermediate analytical tables (per-well AUC, per-compound
ANOVA / LMM p-values, kinetic parameters, normalisation-benchmark
numbers, Monte Carlo iterations, fPCA scores, spatial-correction
diagnostics, logistic-fit summaries, Bayesian posterior summaries,
and the S1–S9 supplementary tables) are **not deposited here**.
They are produced deterministically by running the analysis
scripts on the raw OD600 data, with fixed seeds
(`set.seed(2026)`), so every result is bit-for-bit reproducible
from the code in this repository.

## Reproducibility

### Software environment

- R version 4.5.x (see `environment/sessionInfo.txt` for the exact
  version used for the published analyses).
- All R packages, with versions, are listed in
  `environment/packages.csv`.
- The pipeline is platform-independent; it has been run on
  Windows for the original analyses and is expected to behave
  identically on macOS and Linux.

### Regenerating all figures and tables

From the repository root, with the raw data already in
`data/raw/`:

```r
# 1. Install the gcscreen package (R implementation of the pipeline)
remotes::install_github("mparmol/gcscreen")

# 2. Run the primary pipeline scripts in numerical order
source("code/00_robust_pipeline/P2.1_P2.2_robust_pipeline.R")
source("code/00_robust_pipeline/P2.3_mixed_model.R")
source("code/00_robust_pipeline/P2.4_kinetic_params.R")
source("code/00_robust_pipeline/P2.4_kinetic_figures.R")
source("code/00_robust_pipeline/generate_final_figures.R")
source("code/00_robust_pipeline/generate_fig1_workflow.R")

# 3. Run the validation scripts (independent — any order)
source("code/01_validation/01_zprime_ssmd.R")
source("code/01_validation/02_benchmark_norm.R")
source("code/01_validation/03_monte_carlo.R")
source("code/01_validation/04_functional_data_analysis.R")
source("code/01_validation/05_bscore_vs_spline.R")
source("code/01_validation/06_auc_vs_gompertz.R")
source("code/01_validation/07_bayesian_lmm.R")
```

Each script creates its outputs under `data/intermediate/`
(`primary_pipeline/` for the 00 scripts, `validation/` for the 01
scripts, with a `validation/definitive/` subfolder for the S1–S9
supplementary tables) and writes its PDF figures alongside. The
header of every script documents the exact list of files it
produces.

### Why are the raw data not in the repository?

This is a deliberate choice. The analysis is deterministic given
the deposited code and the seed; raw kinetic recordings of this
size would bloat both the GitHub repository and its Zenodo
snapshot without contributing to method reproducibility.
Researchers who wish to re-run the pipeline on the original raw
data can obtain them by email; researchers who wish to apply the
pipeline to their own screen can do so without any need for our
raw data.

## Eighteen core hits

The dual-convergence pipeline surfaces eighteen reproducible core
hits (BH-FDR < 0.05 in both per-plate ANOVA and library-wide LMM,
after exclusion of Chicago sky blue 6B as a colorimetric artefact
and of Tiratricol as a borderline non-convergent compound in the
final re-run). The list includes canonical positive controls
(cycloheximide, four azole antifungals, ciclopirox, hexachlorophene,
alexidine, thimerosal, thonzonium, merbromin), known antimicrobial
repurposing candidates (amiodarone, nitroxoline, clioquinol,
dequalinium, doxorubicin) and two mechanistically unexpected
molecules (clomiphene citrate, tolcapone). See Table 3 of the
manuscript for the full ranked list. Per-compound posterior
summaries (Bayesian framework) are written to
`data/intermediate/validation/Table_bayesian_hits.txt` on a fresh
run of `code/01_validation/07_bayesian_lmm.R`.
